## 一、目标与约束

### 1.1 目标

把已冻结的设计（H5 v3 + 接口 v2 + Figma v1）落地为可用的生产系统：

| # | 交付物 | 部署形态 |
|---|---|---|
| 1 | H5 客户端 | Cloudflare Pages，主域，邀请制 URL 混淆 |
| 2 | Admin Console | Cloudflare Pages，`admin.*` 子域 |
| 3 | API 服务 | Cloudflare Workers (Hono) |
| 4 | 数据存储 | Neon Postgres + R2 + Upstash Redis |

### 1.2 硬约束（不可妥协）

- **海外部署、不备案、不接审核**（业主 2026-05-28 决策）
- **邀请制 = URL 混淆**：不做登录 / 邀请码，靠不可猜 URL + noindex + 空 robots.txt
- **月成本接近 $0**：按量计费、scale-to-zero、不引入常驻服务
- **大陆 best-effort 可达**：不签 SLA，但需要保留 China-friendly CDN 切换通道
- **未成年模特强制隐去身体数据**：覆盖整体公开策略
- **真名 / 联系方式 / 证件**：永不公开

### 1.3 非目标（明确不做）

为避免开发期反复纠缠，复述业主已决策的不做事项：

- ❌ 客户端登录 / 收藏 / 个性化
- ❌ 客户端在线预约表单（QQ 全程替代）
- ❌ PDF 模卡导出
- ❌ 高级搜索 / 推荐算法 / 虚拟滚动等大数据量优化
- ❌ 微信小程序 / 公众号接入
- ❌ ICP 备案 / 内容审核接入
- ❌ 手机端后台 / 后台 PWA（后置 P3，本期不做）

## 二、代码仓库结构

### 2.1 顶层布局

```
chiyan-web/
├─ apps/
│  ├─ h5/                    # H5 客户端（Cloudflare Pages，主域）
│  ├─ admin/                 # Admin Console（Cloudflare Pages，admin.*）
│  └─ api/                   # API 服务（Cloudflare Workers + Hono）
│
├─ packages/
│  ├─ ui/                    # 共享 shadcn 组件 + Design Tokens
│  ├─ db/                    # Drizzle schema + migrations + 类型导出
│  ├─ types/                 # API 契约（zod schema）+ 共享 TS 类型
│  └─ config/                # tsconfig / eslint / tailwind 共享 preset
│
├─ docs/                     # 设计 + 开发文档（本目录）
├─ .github/workflows/        # CI（lint / typecheck / preview deploy）
├─ pnpm-workspace.yaml
├─ package.json
└─ README.md
```

### 2.2 选型说明

| 决策 | 选项 | 理由 |
|---|---|---|
| 工具链 | pnpm workspace，不引 Turborepo | 三个 app 体量轻，增量缓存收益有限；后期构建变慢再叠加 |
| 共享代码 | `packages/` 而非 git submodule | monorepo 内引用更快、类型贯通 |
| API 契约 | `packages/types` 内放 zod schema | Workers 端做参数校验、Admin 端直接拿到类型；OpenAPI 可从 zod 自动生成 |
| DB schema | `packages/db` 独立 | API + 迁移脚本 + 本地 seed 共用；不放 `apps/api` 内避免循环 |
| UI 库 | `packages/ui` 复用 shadcn 组件 | H5 与 Admin 共享 atoms，但布局 / 业务组件各自归属 app |

### 2.3 现有 `开始设计方案/` 处置

Figma Make 导出的 Vite+React+Tailwind+Radix 代码是参考用，不是最终骨架（见 memory）：

- **保留**：`src/styles/`（tokens 实现）、`src/app/components/`（H5 各 Section 与 ModelCard）
- **重做**：`src/main.tsx` / `App.tsx` 的入口（加 React Router、Tab 切换重写为 URL 路由）
- **不要**的依赖：`@mui/*`、`react-slick`、`canvas-confetti`、`react-dnd`（与 shadcn 体系冲突或冗余）
- **迁移路径**：Phase 2 启动时将组件搬到 `apps/h5/src/`，逐组件接真实数据；旧目录在迁移完成后删除

## 三、技术栈复述

完整选型见 [后台管理系统接口设计方案.md §一.3](./后台管理系统接口设计方案.md)。开发期需特别注意：

| 维度 | 实际约束 |
|---|---|
| Workers 运行时 | V8 Isolate，**禁用** `fs / net / child_process / TCP 持连`，第三方库需 Workers-compatible |
| Neon 驱动 | **必须** 用 `@neondatabase/serverless` HTTP 驱动，不能用 `pg` |
| Redis | **必须** 用 Upstash HTTPS REST，不能用 `ioredis` |
| 鉴权 | 自建 JWT + HttpOnly Cookie + jti 黑名单：`hono/jwt`（HS256 access/refresh/totp_challenge）+ `bcryptjs`（cost 12，依赖 `nodejs_compat` 的 Buffer）+ `otpauth`（TOTP HMAC-SHA1）+ Upstash 存 jti 黑名单与限流。**不引入 Lucia**：JWT + Cookie + jti 黑名单已覆盖接口方案 §3.2 全部要求，省去 ~50KB gzip 与 session store 一层依赖。决策于 Phase 1 Spike A 后回填（commit 8e... 2026-05-28） |
| 加密 | `crypto.subtle`（WebCrypto），不能用 Node `crypto` |
| 登录 | **账密 + TOTP（强制）**，无短信 / 邮件通道；详见 [接口方案 §3.1](./后台管理系统接口设计方案.md) |

### Phase 0 — 工程基建（约 2 工日）

**任务**

1. 仓库初始化：pnpm workspace、`packages/config` 落地（tsconfig / eslint / prettier）
2. 三个空 app 骨架：`apps/h5`（Vite + React 18 + TS）、`apps/admin`（同）、`apps/api`（Hono + Wrangler）
3. Cloudflare account：
   - Pages 项目 × 2（h5 主域、admin 子域）
   - Workers 项目 × 1（api，绑定自定义子域 `api.*` 或同源 `/api/*`）
   - R2 bucket × 2（`chiyan-public` 带 CDN、`chiyan-private` 仅 Workers 访问）
4. 外部账号：Neon project（dev + staging + prod 三 branch）、Upstash Redis × 2（限流 + session）、Sentry org
5. GitHub Actions CI：PR 必须 green（lint + typecheck）；main 自动 deploy 到 staging
6. DNS 切到 Cloudflare，主域 + admin 子域占位页面上线

**退出标准**

- `pnpm dev` 在三个 app 都能起本地
- `pnpm typecheck` / `pnpm lint` 全 green
- 三个域名访问到对应占位页（HTTPS 自动签发）
- CI 通过

**风险**

- Cloudflare account 归属（业主 or 开发方）影响后续移交，**必须先和业主确认**
- 域名 DNS 移交（业主已申请，需要在 Phase 0 拿到管理权）

### Phase 1 — 基础设施层（约 3 工日）

**任务**

1. `packages/db`：按 [接口方案 §二](./后台管理系统接口设计方案.md) 实体清单写 Drizzle schema + 初始 migration
2. `packages/types`：用 zod 写公开 + 后台所有接口的请求 / 响应 schema，导出 TS 类型
3. API 基座：
   - Hono 应用骨架、路由分组（`/auth`、`/admin`、`/public`）
   - 统一响应 / 错误格式（含 `request_id`）
   - 请求日志（结构化 JSON，输出到 Cloudflare Logs）
4. 鉴权（自建 JWT + Refresh Cookie + 强制 TOTP，不用 Lucia，见 §三技术栈表）：
   - `/auth/login`（账密）→ `/auth/login/totp`（TOTP code）两步登录
   - 首登强制流程：`must_change_password` → `/auth/change-password` → `totp_enrolled=false` → `/auth/totp/setup` + `verify` → 正式放行
   - 一次性密码生成（20 字符 URL-safe random，明文仅响应一次，bcrypt cost 12 落库）
   - 失败锁定：连续 5 次失败 → 账号锁定 15 min；Owner 可 `/admin/accounts/:id/unlock`
   - jti 黑名单（Upstash）
   - HttpOnly + Secure + SameSite=Lax Cookie
   - CSRF 双 Token 校验
5. 限流中间件（Upstash 滑动窗口，按 [§七.1](./后台管理系统接口设计方案.md) 配置）
6. 加密工具（real_name_enc / totp_secret_enc，AES-256-GCM via WebCrypto，密钥版本化）
7. 一次性密码 / TOTP secret **不进日志 / Sentry / 审计 payload** 的统一脱敏中间件

**退出标准**

- Owner 在 Admin 后台创建新账号 → 拿到一次性密码 → 用其首登 → 强制改密 → 绑定 TOTP → 进入主界面，全程闭环
- `/auth/login` + `/auth/login/totp` + `/auth/me` + `/auth/logout` 全部 200，错码符合 [§五.3](./后台管理系统接口设计方案.md)
- 连续 5 次错密触发 `locked_until`，Owner 可手动解锁
- 限流触发返回 429 + Retry-After
- DB migration 在 dev / staging 都跑通
- 加密字段往返一致（写入 → 读取 → 解密 = 原文），密钥版本号正确记录
- grep 日志确认明文密码 / TOTP secret 0 命中

**风险**

- 自建 JWT + Drizzle + Neon HTTP 组合需要 spike：bcryptjs CPU 消耗、otpauth Workers 兼容、neon-http 关系型查询与 batch 多语句边界（已在 Phase 1 Step 1 Spike A 全部跑通：5 lib，24 tests green）
- TOTP 时间漂移：服务器与客户端时间偏差 > 30s 会导致验证失败，需启用 `window=1`（容忍前后 1 个时间窗）

### Phase 2 — 公开 API + H5 客户端（约 5 工日）

**任务**

1. 公开 API 实现：
   - `GET /public/today?date=`
   - `GET /public/models?...`
   - `GET /public/models/:code`
   - `GET /public/studio-info`
   - `POST /public/track`
2. CDN 缓存策略：按 [§四.9](./后台管理系统接口设计方案.md) 设置 `Cache-Control`；数据变更触发 `POST /zones/<id>/purge_cache`
3. H5 客户端：
   - 把 `开始设计方案/` 的组件迁到 `apps/h5/src/`，去除 mock 数据
   - 4 个 Tab + ModelDetailSheet 接入真实 API
   - 路由：URL `/`、`/today`、`/roster`、`/contact`、`/m/:code`（详情独立 URL 便于分享）
   - 顶部 Large Title → Compact 滚动过渡
4. 状态屏齐备（见 [H5 §五](./模特资料与当日通告接单-H5设计方案.md)）：加载骨架、空、错、离线、当日休息
5. 分享通道实现：
   - 微信 JSSDK（需公众号配置，暂用工作室公众号 or 业主决策方案）
   - QQ 互联 JSAPI
   - Web Share API + 兜底 Sheet
6. QQ 接单链路：复制话术 + 多 scheme 唤起 + 场景分支兜底（[H5 §八.5](./模特资料与当日通告接单-H5设计方案.md)）
7. SEO 反向控制：`noindex` meta + 空 robots.txt + 移除 OG 预览

**退出标准**

- H5 staging 链路完整：首页 → 当日通告 → 模特详情 → QQ 接单 Toast
- 公开 API P95 < 300ms（Cloudflare 边缘节点实测）
- iPhone 14 Pro + iPhone SE 双尺寸 + 微信内置浏览器 + QQ 内置浏览器 四端走查通过
- Lighthouse Performance ≥ 85（移动端）

**风险**

- 微信 JSSDK 需要业主公众号配置 + 域名加白，**Phase 2 启动前必须解决**
- 设计稿 Day 9 评审若晚于 Phase 2 启动，先做接口对接，UI 留 buffer

### Phase 3 — Admin Console 核心模块（约 6 工日）

**任务**

1. Admin 骨架：shadcn 布局 + 侧边栏 + 主区域 + 路由（React Router）

4. 当日名单模块（最高频操作）：

   - 一键复制昨日
   - 时间预设备注
   - 历史回看

   - R2 直传（`/admin/media/sign` → 前端 PUT → `/admin/media/register`）

   - 缩略图 + 原图分离展示

**退出标准**

- **业主可在 staging 独立完成核心动线**：登录 → 新增模特 → 上传 5 张图（含水印生成）→ 加入今日名单 → H5 端 5 分钟内看到

### Phase 5 — 性能 / 安全 / 运维（约 3 工日）

**任务**

   - CSP / HSTS / X-Frame-Options / Referrer-Policy
   - 文件 magic number 校验
   - SQL injection（Drizzle 参数化）/ XSS（输出转义、富文本 sanitize）

   - Sentry 接入（H5 + Admin + Workers）

4. 备份策略：
   - Neon PITR 7 天 + 周快照人工归档
   - R2 Object Versioning + 7 天保留
   - 审计日志 1 年内热查 + 超期冷归档
5. 告警阈值（错误率 > 1% / 短信余额 / R2 流量 / Neon CU）

**退出标准**

- Lighthouse 三端（H5 / Admin / API 模拟）跑分达标
- Sentry 接 0 error（staging 1 周观察期）
- 主动 Pull-the-plug 测试：杀 Neon / Upstash / R2 任一，系统应有清晰错误提示而非崩溃

## 七、风险与备选

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 大陆访问 best-effort 不达预期 | 中 | 中 | 预留 China-friendly CDN（CDN77 / BytePlus / EdgeNext）切换；Cloudflare DNS 后端切换无业务变更 |

| 一次性密码线下交付泄漏 | 低 | 中 | 强制首登改密 + 强制 TOTP；即使密码被偷看，无 TOTP 仍登不进；密码 history 5 条避免回滚弱密码 |
| Workers + Neon HTTP + 自建鉴权组合落地坑 | 中 | 中 | Phase 1 Step 1 Spike A 已跑通 crypto/jwt/totp/password 4 个 lib（24 tests green）；neon-http + Drizzle 关系型查询与 batch 边界放到 Step 7 等账号到位后再 spike |
| `@cf/image-resizing` 水印效果不理想 | 中 | 低 | 备选：用 squoosh-cli 或 sharp（在外部 worker）异步处理；水印做不精细就改为 SVG overlay |
| 业主品牌 VI 拖延 | 高 | 低 | 已 token 化，上线前任意时间替换 |
| 设计 / 开发完全串行导致延期 | 中 | 高 | 严格按评审节点章节级冻结；开发可先用占位 UI 跑通数据流 |
| Cloudflare 海外节点突发故障 | 低 | 高 | Sentry 告警 + 静态降级页（R2 直挂） |
| 单人全栈 burnout / 病假 | 中 | 高 | 文档先行（接口契约 + schema 锁死），任何一段都可被外部接手 |
