## 一、项目定位

### 1. 核心用途

面向**品牌方、经纪人、制片、活动策划**等甲方客户，移动端轻量化展示自有签约模特全档案，实时更新**当日可接通告模特名单**，替代传统 Word + 图片打包发送模式，实现：

- 一屏看档 — 当日可出工模特一目了然
- 一键选模 — 标签筛选 + 详情即看即决
- 一键咨询 — 直达 QQ，自动复制带模特编号的预约话术

### 2. 设计调性

时尚高级感 × iOS 原生感的平衡：

- **iOS 一面**：克制的层级、克制的动效、克制的色彩；遵守安全区、Dynamic Type、原生手势预期
- **时尚一面**：电影感图片调色、低饱和高级灰、香槟金点缀、留白克制但充足

不做的事：渐变彩色按钮、拟物阴影、Material Design 浮动球、3D 卡片倾斜、过度的微交互。

### 3. 核心优势

- 移动端原生感强，iOS 用户上手零成本；
- 当日通告区与全库强区隔，决策路径短；
- 模特资料结构化呈现，符合行业选模 SOP；
- QQ / Safari / Chrome 多场景秒开，可转发、可收藏到主屏（PWA-lite）。微信内置浏览器不在适配范围。

## 二、iOS 设计语言对齐（v2 新增章节）

### 1. 字体系统

| 用途 | 字体 | 说明 |
| --- | --- | --- |
| 英文 / 数字 | SF Pro Display / SF Pro Text | iOS 原生字体，系统自动 fallback |
| 中文 | PingFang SC | iOS 中文默认，Android fallback 到 思源黑体 / 系统默认 |
| 字号锚点 | 11 / 13 / 15 / 17 / 22 / 28 / 34 | 对齐 iOS Type Scale；正文 17pt，标题 28/34pt |

**强制要求**：支持 Dynamic Type（用户在系统设置中放大字号时，页面随之缩放，不破版）。

### 2. 圆角与形状

- 全局采用 **iOS 连续圆角（continuous corner，俗称"squircle"）** — `border-radius` 配合 `corner-smoothing` 或 SVG mask 实现，视觉上比标准圆角更柔和
- 卡片：20pt
- 按钮：12pt（小） / 14pt（中） / 28pt 全圆（胶囊按钮）
- Sheet 顶部：10pt 仅上两角

### 3. 颜色系统（亮 / 暗双套）

> ⏳ **业主品牌 VI 设计中**：本节色板为方案默认值，待 logo / 标准色确定后整体替换；香槟铜可平滑切换为玫瑰金 / 银灰 / 黄铜 / 墨绿等高端色系，结构不变。

#### 亮色模式

| 角色 | 色值 | 用途 |
| --- | --- | --- |
| Background | #F7F7F8 | 全局底色 |
| Surface (card) | #FFFFFF | 卡片 |
| Label / Primary text | #1C1C1E | 主文本 |
| Secondary text | #6C6C70 | 副文本 |
| Tertiary text | #AEAEB2 | 辅助文本 |
| Separator | rgba(60,60,67,0.12) | 分割线 |
| Accent (品牌) | #8C6A4B 香槟铜 | 选中、链接、强调 |
| Critical (重点) | #A33B3B 浅酒红 | 当日可接标签、预约 CTA |
| Success | #2E7D5B | 档期可约 |

#### 暗色模式

| 角色 | 色值 |
| --- | --- |
| Background | #000000 |
| Surface (card) | #1C1C1E |
| Label | #FFFFFF |
| Secondary | #98989F |
| Separator | rgba(84,84,88,0.65) |
| Accent | #B89070 |
| Critical | #C76060 |

**严禁**：高饱和红、亮蓝、亮绿、霓虹紫等任何破坏高级感的色彩。

### 4. 间距与栅格

- 基础间距单位：4pt（4 / 8 / 12 / 16 / 20 / 24 / 32 / 40）
- 安全边距：左右 16pt，重要内容区 20pt
- 卡片之间纵向间距：12pt
- 顶部安全区：`env(safe-area-inset-top)`
- 底部安全区 + Tab Bar 高度：`env(safe-area-inset-bottom) + 56pt`

### 5. 动效原则

- 全局缓动函数：`cubic-bezier(0.32, 0.72, 0, 1)`（iOS 标准 spring 曲线近似）
- 过渡时长：200–350ms，超出 400ms 视为过重
- Sheet 弹出：spring 阻尼 0.8，质量 1
- 点击反馈：scale 0.97 + 不透明度 0.85，时长 120ms
- 触觉反馈（条件支持）：`navigator.vibrate(10)` 模拟轻触反馈；iOS Safari 不支持时静默降级

### 6. 图标系统

- 优先使用 **SF Symbols 风格** 的图标（线性、统一描边 1.5pt）
- 自有图标库以 SVG 提供，命名对齐 SF Symbols（`person.crop.circle`、`calendar.badge.clock` 等）
- 不混用 Material Icons / FontAwesome

## 四、详细页面设计

### （一）顶部导航

- **样式**：iOS 大标题模式（Large Title）。页面顶部不固定，向下滚动时大标题收缩为标准导航栏（Compact Title + 毛玻璃背景 `backdrop-filter: blur(20px)`）。
- **右上角**：图标按钮 `square.and.arrow.up`（分享），唤起原生分享面板：QQ 好友 / QQ 空间 / iOS 系统分享（AirDrop、备忘录、复制链接等），详见 §八.4。**不接微信分享**（业主决策）。
- **不再设置全局悬浮咨询按钮**——咨询入口下沉到 Tab Bar 第 4 项 + 详情页底部固定 CTA，避免遮挡内容与 Safari 底栏冲突。

### （二）首屏 Hero Banner

- 16:9 比例（不强行全屏，避免 iPhone 各机型差异）
- 背景：低饱和高级灰 + 模特群像虚化底图 + 顶部到底部 10% 暗角
- 文案层级：
  - 主标题（28pt，SemiBold）：XX 模特工作室
  - 副标题（15pt，Regular，二级文本色）：当日通告 · 即刻可约
  - 标语行（13pt，Accent 色）：每日更新在班模特 · 真实档期
- 装饰：**取消** v1 的几何线条元素，保持极简

### （三）核心板块：当日可接通告

#### 1. 板块标题（自动日期）

格式：`2026年5月25日 · 周一 · 在班模特`

- 字号：22pt SemiBold
- 副行：`共 12 位 · 9:00–22:00 可联系经纪人确认`（13pt Secondary）

#### 2. 筛选交互（iOS 标准组件）

- **Segmented Control**（工种）：全部 / 平面 / 电商 / 走秀 / 短视频 / 礼仪
  - 选中底色 #FFFFFF（亮）/ #3A3A3C（暗），未选中透明
  - 整体外框胶囊，背景 #EFEFF0
- **风格 Chips** 横向滚动：御姐 · 清纯 · 港风 · 高级脸 · 甜妹 · 中性 · 异国感
  - 多选，选中态填充 Accent
  - 滚动条隐藏，左右淡出遮罩
- 筛选结果实时刷新（无加载态打断），数量变化以 spring 动画过渡

#### 3. 卡片布局

- **2 列网格**（iPhone SE 至 Pro Max 均适配）
- 卡片宽度：`(viewport - 16*2 - 12) / 2`
- 卡片间距：横向 12pt，纵向 12pt

#### 4. 单张卡片信息（业主决策：公开展示）

```
┌─────────────────────┐
│                     │
│   [主图 1:1.25]     │  ← 高度略大于正方形，更显腿长
│                     │
│  ●可接 ●异地        │  ← 左下角浮层标签（毛玻璃底）
├─────────────────────┤

│  175 · 48kg · 上海   │  13pt Secondary
│  88-60-88 · 38码     │  11pt Tertiary
│  #港风 #高级脸       │  11pt Accent Chips
└─────────────────────┘
```

**字段策略（业主决策：完全公开）**：
- **直出**：化名、身高、体重、三围、鞋码、常驻地、风格标签
- **隐藏**：真实姓名（统一以化名 / 艺名展示）、本人联系方式
- **整卡可点击** 进入详情；详情底部统一 CTA，避免双按钮误触
- **未成年模特** 自动隐去身体数据（§七 硬性规则覆盖）

### （四）次要板块：全部模特库

- 顶部 iOS Search Bar（带"取消"按钮的标准样式）
- Segmented Control 切换：按风格 / 按工种
- 卡片与当日通告共用组件，仅去除"当日在岗"角标
- 空状态：插画占位（极简单色线稿）+ 文案"更多模特持续更新中"
- 长列表：使用 `IntersectionObserver` 懒加载，下拉触发刷新动画（iOS 标准 spinner）

### （五）模特详情（iOS Sheet）

#### 1. 呈现方式

- **半屏 Sheet**（medium detent，约屏幕 60%）→ 可上滑展开至 large detent（约 92%）
- 顶部 grabber 指示条（36×4pt，圆角）
- 下滑超过阈值关闭，背景遮罩可点击关闭
- 关闭后保留滚动位置（不重置列表）

#### 2. 内容结构（从上至下）

1. **轮播图**（16:9，分页指示器为 iOS 小圆点）— 形象、作品、花絮
2. **基本档案**（卡片化，全字段直出）
   - 化名 / 艺名 · 风格标签
   - 身高 / 体重 / 三围 / 鞋码 / 年龄段 / 籍贯 / 常驻地
   - 业主决策：全部公开展示，无需二次点击；图片仍保留水印防外传
3. **接单范围**：可承接类型 + 异地 + 档期偏好
4. **代表合作**（横向滚动卡片，品牌 logo + 项目名 + 年份）
5. **档期状态**：今日 / 本周 / 下周（绿点 = 可约，灰点 = 已满，斜线 = 待确认）
6. **完整模卡** 入口：跳转到独立模卡页（高清大图，禁右键 / 加水印）

#### 3. 底部固定 CTA

胶囊按钮 + 主次结构：
- 主按钮（Accent 填充）：**预约该模特** → 自动复制 QQ 号（含模特编号 + 预约话术）+ 唤起 QQ App（详见 §八.5）
- 次按钮（Outline）：**分享该模特** → 唤起原生分享面板（详见 §八.4，业主决策不做 PDF 导出）

### （六）底部 Tab Bar & Footer

- Tab Bar：4 项，毛玻璃背景，分割线 1px hairline
- Footer 仅出现在「联系」Tab 内，不全局占位：
  - 工作室一句话简介
  - **QQ**（主接单入口，大号显示 + 复制按钮 + 二维码）/ 电话 / 地址
  - 点击 QQ：复制 QQ 号 + 尝试唤起 QQ App；点击电话：直接呼起拨号
  - 版权信息

## 七、隐私、合规与版权（v2 新增）

> **业主决策（2026-05-25）**：采用「公开展示」策略 — 三围、体重、鞋码等行业基础数据直出，便于客户初筛与选模。

1. **肖像权**：所有图片需有模特签署的肖像使用 + 网络展示授权；上线前由经纪人统一回收授权书
2. **字段分级（业主决策后）**：
   - **完全公开**（卡片 + 详情均展示）：化名、身高、体重、三围、鞋码、年龄段、籍贯、常驻地、风格标签、可接范围、合作履历
   - **永不公开**：真实姓名、本人联系方式（手机 / 私人 QQ）、家庭住址、身份证件
   - 接单后由工作室经纪人通过 QQ 提供必要补充信息
3. **防盗图**：关键大图叠加视觉水印（半透明工作室 logo + 模特编号），禁用右键 / 长按保存（iOS Safari 仅能降低难度，无法 100% 阻止）
4. **未成年保护（硬性规则，覆盖公开策略）**：未成年模特必须监护人书面授权 + **强制不展示三围 / 体重等身体数据**，即使整体策略是公开
5. **数据合规**：访客埋点若涉及个人信息，需在底部提供「隐私说明」入口；模特本人有权随时申请下架其资料（后台一键下架，详见 §九 后台管理）
6. **AI 生成声明**：若模特图片含 AI 修图 / 生成内容，按平台合规要求标注

## 八、技术与适配方案

### 1. 适配

- iOS Safari 15+、Chrome on Android、QQ 内置浏览器。微信内置 X5 浏览器不在适配范围，命中时降级到引导卡（见 §六）
- 安全区：`viewport-fit=cover` + `env(safe-area-inset-*)`
- 横竖屏：以竖屏为主，横屏降级展示（非首要场景）
- 折叠屏 / 大屏 iPad：单列布局保持卡片不过宽（max-width 600pt）

### 2. 图片策略

- 格式：WebP 优先，AVIF 渐进升级，jpg 兜底
- 尺寸：`srcset` 提供 1x / 2x / 3x；卡片缩略 400w，主图 800w，模卡 1600w
- 占位：blurhash 或 LQIP（10×15 模糊缩略）
- 存储 / CDN：Cloudflare R2（egress 免费） + Cloudflare Images / Workers `@cf/image-resizing`（按需缩放 / WebP / 水印）
- 懒加载：`loading="lazy"` + IntersectionObserver 提前 200px

### 3. 性能预算

| 指标 | 目标 |
| --- | --- |
| 首屏 LCP | < 2.0s（4G） |
| 总包体积（首屏） | < 200KB gzip |
| 首屏图片总和 | < 500KB |
| 交互响应 INP | < 200ms |

### 4. 分享（QQ + iOS 原生 双通道；微信不接，业主 2026-05-29 决策）

| 通道 | 实现 | 覆盖渠道 |
| --- | --- | --- |
| QQ 内置浏览器 | QQ 互联 JSAPI（`mqq.ui.shareMessage` / `shareToQzone`） | QQ 好友 / QQ 空间 |
| iOS Safari & Android Chrome | `navigator.share()` Web Share API | 系统已安装的所有 App：QQ、邮件、AirDrop、备忘录、复制链接… |
| 全场景兜底 | 自绘底部 Sheet（iOS 风格） | 复制链接 / 保存图片 / 二维码 |
| 微信内置浏览器 | 不适配 | 显示「请在 Safari / Chrome / QQ 中打开」引导卡，不调用任何微信 API |

- **卡片粒度分享**：每位模特可单独分享，URL 携带 `?model=<编号>` 直达详情
- **分享卡片**：自动生成封面（模特主图 + 工作室签名 + 编号 + 二维码），统一品牌露出
- **唤起失败**：任何通道调用失败时，自动降级到兜底 Sheet，不阻断用户

### 5. 接单链路（QQ 全程）

#### 5.1 操作流程

1. 用户点击「预约该模特」/ 联系 Tab 内的 QQ 入口
2. 调用 `navigator.clipboard.writeText()` 复制预约话术，模板：
   ```
   您好，咨询当日通告模特：林夕（编号 M-2026-0125），档期：今日。
   工作室 QQ：xxxxxx
   ```
3. 同时尝试唤起 QQ App（多 scheme 容错）：
   - iOS：`mqq://im/chat?chat_type=wpa&uin=<QQ号>&version=1`
   - Android：`mqqwpa://im/chat?chat_type=wpa&uin=<QQ号>`
   - PC / Web 兜底：`https://wpa.qq.com/msgrd?v=3&uin=<QQ号>&site=qq&menu=yes`
4. 顶部 Toast 提示「QQ 号已复制，正在打开 QQ…」（毛玻璃风格，3 秒自动消失）

#### 5.2 场景分支

| 场景 | 行为 |
| --- | --- |
| iOS Safari | scheme 唤起：QQ 已装则跳转，未装则保留复制结果 + 引导下载 |
| 微信内置浏览器 | 不适配 → 显示「请在 Safari / Chrome / QQ 中打开」引导卡，不触发 scheme |
| QQ 内置浏览器 | 直接调用内部跳转 API，无需手动复制 |
| Android Chrome / 其他 | 同 iOS Safari 流程，scheme 切换为 `mqqwpa://` |
| 桌面 Web | 卡片同时显示大号 QQ 号 + 复制按钮 + QQ 二维码（手机扫码加好友） |

#### 5.3 话术拼接规范

- **自动注入**：模特编号、化名、所选档期标签、工作室 QQ
- **不暴露**：真实姓名、三围、电话等敏感字段
- **末尾签名**：追加一行工作室简介，便于客户群转发后仍可识别来源

#### 5.4 防滥用

- 同一设备 60 秒内最多触发 5 次复制 / 唤起，避免脚本刷量
- 后台埋点：记录哪位模特被点击预约最多，反哺当日推荐顺序与卡片排序权重

## 十一、整体体验总结

v2 在保留 v1 业务直觉的基础上，做了三件事：

1. **把"高端"翻译成可执行的 iOS 设计规范** — 不再是形容词，而是字号、圆角、动效曲线、安全区等可验收的指标
2. **把"实用"翻译成完备的状态机** — 加载 / 空 / 错 / 离线 / 当日休息全部覆盖
3. **把"合规与隐私"补齐** — 模特行业是肖像 / 个人信息密集场景，v1 完全没提，这是必须补的功课

落地后，预期可在 iPhone 上获得接近原生 App 的浏览体验，同时保留 H5 即开即用、易转发的核心优势。
