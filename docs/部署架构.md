## 〇、本文档是什么

API 设计文档定义"接口契约与数据模型"，本文档定义"机器上长什么样、谁起谁停、坏了怎么修"。回答：

- VPS 拿到后第一天怎么初始化（OS / 用户 / 防火墙 / 软件源）
- 每个服务对应的 systemd unit、Caddyfile、环境变量从哪里来
- 2G RAM 怎么挤——内存预算、关键调优参数
- CI/CD 流水线（GitHub Actions → SSH → systemctl restart）
- 备份与恢复（pg_dump + rsync → Backblaze B2）
- 灾难场景的应对清单

本文档**不替代**实操运维手册，但任何变更（端口 / 路径 / 内存预算 / 启停依赖）必须先回写本文档再上机。

## 二、初始化清单（首装一次）

```bash
# 1. 基础环境
apt update && apt upgrade -y
apt install -y curl gnupg2 ca-certificates lsb-release ufw fail2ban unattended-upgrades

# 2. 用户与目录（最小权限）
adduser --system --group --home /var/chiyan chiyan
mkdir -p /var/chiyan/{media,logs,backups,run} /etc/chiyan
chown -R chiyan:chiyan /var/chiyan
chmod 750 /var/chiyan /etc/chiyan

# 3. 防火墙
ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp           # SSH（建议改非标端口 + key-only）
ufw allow 80/tcp           # Caddy HTTP（自动 redirect HTTPS）
ufw allow 443/tcp          # Caddy HTTPS
ufw enable

# 4. 自动安全更新（仅 security 类）
dpkg-reconfigure -plow unattended-upgrades

# 5. Node.js 22 LTS（NodeSource）
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs
npm install -g pnpm

# 6. PostgreSQL 16(PGDG 源)
sh -c 'echo "deb https://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
curl -fsSL https://www.postgresql.org/media/keys/ACCC4CF8.asc | gpg --dearmor -o /etc/apt/trusted.gpg.d/postgresql.gpg
apt update
apt install -y postgresql-16

# 7. Redis 7
apt install -y redis-server

# 8. Caddy 2
apt install -y debian-keyring debian-archive-keyring apt-transport-https
curl -fsSL https://dl.cloudsmith.io/public/caddy/stable/gpg.key | gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -fsSL https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt | tee /etc/apt/sources.list.d/caddy-stable.list
apt update
apt install -y caddy

# 9. sharp 原生依赖
apt install -y libvips libvips-dev

# 10. 备份工具
apt install -y rclone gzip rsync
```

> `fail2ban` 拦 SSH 暴力破解；`unattended-upgrades` 只接 `-security` 通道，避免 minor 升级抖动。

## 三、systemd 单元

四个核心 unit。所有 unit 加 `MemoryMax`、`Restart=on-failure`、`User=` 与 `PrivateTmp=true`。

### 3.1 `/etc/systemd/system/chiyan-api.service`

```ini
[Unit]
Description=ChiYan API (Hono on Node)
After=network-online.target postgresql.service redis-server.service
Wants=network-online.target
Requires=postgresql.service redis-server.service

[Service]
Type=simple
User=chiyan
Group=chiyan
WorkingDirectory=/var/www/chiyan/apps/api
EnvironmentFile=/etc/chiyan/secrets.env
Environment=NODE_ENV=production
Environment=NODE_OPTIONS=--max-old-space-size=384
ExecStart=/usr/bin/node dist/index.js
Restart=on-failure
RestartSec=3
MemoryMax=512M
LimitNOFILE=8192

# 安全沙箱
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=true
PrivateTmp=true
ReadWritePaths=/var/chiyan/media /var/chiyan/logs /var/chiyan/run
ProtectKernelTunables=true
ProtectKernelModules=true
ProtectControlGroups=true

[Install]
WantedBy=multi-user.target
```

> `EnvironmentFile=/etc/chiyan/secrets.env` 内放 `DATABASE_URL` / `REDIS_URL` / `JWT_PRIVATE_KEY` / `AES_KEY_V1` / `SENTRY_DSN` / `CLOUDFLARE_API_TOKEN`，权限 `0600 root:chiyan`。

### 3.2 `/etc/postgresql/16/main/postgresql.conf` 关键覆盖

```conf
listen_addresses = '127.0.0.1'
max_connections = 20
shared_buffers = 256MB
effective_cache_size = 1GB
work_mem = 8MB
maintenance_work_mem = 64MB
wal_buffers = 8MB
random_page_cost = 1.1
effective_io_concurrency = 200
checkpoint_completion_target = 0.9
log_min_duration_statement = 500ms   # 慢查询告警阈值
```

`pg_hba.conf` 仅保留 `local` + `127.0.0.1/32` 走 `scram-sha-256`。

### 3.3 `/etc/redis/redis.conf` 关键覆盖

```conf
bind 127.0.0.1
port 6379
protected-mode yes
maxmemory 128mb
maxmemory-policy allkeys-lru
save ""                          # 关 RDB
appendonly yes                   # 开 AOF
appendfsync everysec
requirepass <redis-password>     # 写入 secrets.env REDIS_URL
```

### 3.4 Caddy（`/etc/caddy/Caddyfile`）

```caddy
{
    email ops@chiyan.com
    servers {
        protocols h1 h2 h3
    }
    log {
        output file /var/chiyan/logs/caddy-access.log {
            roll_size 100mb
            roll_keep 14
        }
        format json
    }
}

# Cloudflare Origin CA 证书（15 年）放在 /etc/caddy/cf-origin.pem + .key
(tls_cf) {
    tls /etc/caddy/cf-origin.pem /etc/caddy/cf-origin.key {
        protocols tls1.2 tls1.3
    }
}

# H5 主域
chiyan.com {
    import tls_cf

    root * /var/www/chiyan-h5/dist
    encode zstd gzip
    try_files {path} /index.html
    file_server

    # 媒体直发本地磁盘（仅公开层 gallery/thumbs/cover；originals 由 API 控制）
    handle /media/* {
        root * /var/chiyan
        file_server
        header Cache-Control "public, max-age=86400, immutable"
    }

    header {
        Strict-Transport-Security "max-age=31536000; includeSubDomains; preload"
        X-Frame-Options DENY
        X-Content-Type-Options nosniff
        Referrer-Policy same-origin
        # noindex 在 HTML head <meta> 已注入，这里追加 X-Robots-Tag 兜底
        X-Robots-Tag "noindex, nofollow, noarchive"
    }
}

# Admin 子域（IP 白名单 + 同时仍要登录 + TOTP）
admin.chiyan.com {
    import tls_cf

    @whitelist remote_ip 1.2.3.4/32 5.6.7.8/32   # 运维 + 主理人办公出口
    handle {
        @blocked not remote_ip 1.2.3.4/32 5.6.7.8/32
        respond @blocked 403
    }

    root * /var/www/chiyan-admin/dist
    encode zstd gzip
    try_files {path} /index.html
    file_server

    header {
        Strict-Transport-Security "max-age=31536000; includeSubDomains"
        X-Frame-Options DENY
        Content-Security-Policy "default-src 'self'; img-src 'self' https://chiyan.com data:; script-src 'self'; style-src 'self' 'unsafe-inline'; connect-src 'self' https://api.chiyan.com"
    }
}

# API 子域
api.chiyan.com {
    import tls_cf

    reverse_proxy 127.0.0.1:3000 {
        header_up X-Real-IP {remote_host}
        header_up X-Forwarded-Proto {scheme}
    }

    header {
        Strict-Transport-Security "max-age=31536000"
        X-Content-Type-Options nosniff
    }
}
```

> Cloudflare Origin CA 证书在 CF Dashboard 一次性签发 15 年，避免 Let's Encrypt 频繁续期 + 出向带宽。
> `chiyan.com` 之所以是不可猜的 URL，是因为业主决定走 invitations-by-obscurity（详见 [H5 方案 §〇](./模特资料与当日通告接单-H5设计方案.md)），实际域名 / URL 前缀以 1Password 共享条目为准，不入仓库。

## 四、目录布局

```
/var/chiyan/
  ├─ media/                    # 后台上传产物
  │   ├─ models/<id>/
  │   │   ├─ cover.webp / .avif
  │   │   ├─ gallery/          # 水印副本，公开
  │   │   ├─ thumbs/           # 200w / 400w / 800w，公开
  │   │   └─ originals/        # 无水印原图，仅后台 Token 可读
  │   └─ studio/               # 工作室通用素材
  ├─ logs/                     # 应用 + Caddy 日志
  ├─ backups/                  # 本地备份临时区，凌晨上传到 B2 后清空
  └─ run/                      # 进程 PID / Socket

/var/www/chiyan/               # git 工作树（本仓库）
/var/www/chiyan-h5/dist        # pnpm --filter h5 build 产物
/var/www/chiyan-admin/dist     # pnpm --filter admin build 产物

/etc/chiyan/
  └─ secrets.env               # 0600 root:chiyan，systemd EnvironmentFile

/etc/caddy/
  ├─ Caddyfile
  ├─ cf-origin.pem
  └─ cf-origin.key
```

> `/var/chiyan/media/models/<id>/originals/` 的目录 mode 设 `0700`，文件 mode `0600`，owner `chiyan:chiyan`，systemd `ReadWritePaths` 只挂这个分支；Caddy 进程的 `User=caddy` 完全访问不到原图。

## 五、CI / CD

### 5.1 流水线（GitHub Actions）

```yaml
name: deploy

on:
  push:
    branches: [main]

concurrency:
  group: deploy-prod
  cancel-in-progress: false   # 部署不抢占

jobs:
  deploy:
    runs-on: ubuntu-24.04
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v3
        with: { version: 9 }

      - uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: pnpm

      - run: pnpm install --frozen-lockfile
      - run: pnpm -r typecheck
      - run: pnpm -r test
      - run: pnpm -r build

      # rsync 产物到 VPS，原子切换软链
      - name: Deploy via SSH
        env:
          SSH_KEY: ${{ secrets.DEPLOY_SSH_KEY }}
          SSH_HOST: ${{ secrets.DEPLOY_HOST }}
          SSH_USER: deploy
        run: |
          mkdir -p ~/.ssh && echo "$SSH_KEY" > ~/.ssh/id_ed25519
          chmod 600 ~/.ssh/id_ed25519
          ssh-keyscan $SSH_HOST >> ~/.ssh/known_hosts

          # 1. API: 推 dist + node_modules + package.json
          rsync -az --delete \
            apps/api/dist/ apps/api/package.json apps/api/node_modules/ \
            $SSH_USER@$SSH_HOST:/var/www/chiyan/apps/api-next/

          # 2. 前端 dist
          rsync -az --delete apps/h5/dist/ \
            $SSH_USER@$SSH_HOST:/var/www/chiyan-h5/dist-next/
          rsync -az --delete apps/admin/dist/ \
            $SSH_USER@$SSH_HOST:/var/www/chiyan-admin/dist-next/

          # 3. 原子切换
          ssh $SSH_USER@$SSH_HOST 'bash -s' <<'EOF'
          set -euo pipefail
          ln -sfn /var/www/chiyan/apps/api-next /var/www/chiyan/apps/api
          ln -sfn /var/www/chiyan-h5/dist-next /var/www/chiyan-h5/dist
          ln -sfn /var/www/chiyan-admin/dist-next /var/www/chiyan-admin/dist
          sudo systemctl restart chiyan-api
          sudo systemctl reload caddy
          EOF

      # 健康检查
      - name: Smoke test
        run: |
          for i in {1..10}; do
            curl -fsS https://api.chiyan.com/health && exit 0
            sleep 3
          done
          exit 1
```

> `deploy` 用户在 VPS 上配 `sudoers`，仅授权 `systemctl restart chiyan-api`、`systemctl reload caddy` 两个命令，无 root 权限。

### 5.2 数据库迁移

- 走 Drizzle Kit，迁移脚本在 `packages/db/migrations/`
- CI 在 `pnpm -r build` 后执行 `pnpm --filter db migrate:deploy`，目标是 staging；prod 迁移**人工触发**（GitHub Actions `workflow_dispatch`）
- 任何 destructive migration（drop column / drop table）拆两次发布：先发兼容版本上线观察，下次再 drop

### 5.3 回滚

- 软链回旧版本：`ln -sfn /var/www/chiyan/apps/api-prev /var/www/chiyan/apps/api && systemctl restart chiyan-api`
- 数据库回滚靠**前向兼容设计** + `pg_dump` 兜底；不写 down migration（容易出错）

## 六、备份与恢复

### 6.1 每日备份脚本（`/usr/local/bin/chiyan-backup.sh`）

```bash
#!/usr/bin/env bash
set -euo pipefail

DATE=$(date +%Y%m%d-%H%M)
BACKUP_DIR=/var/chiyan/backups
B2_REMOTE=b2:chiyan-backups

# 1. 数据库（custom format，便于 pg_restore）
sudo -u postgres pg_dump -Fc -d chiyan_prod \
  | gzip > "$BACKUP_DIR/db-$DATE.dump.gz"

# 2. 媒体增量
rclone sync /var/chiyan/media "$B2_REMOTE/media/" \
  --transfers 4 --checkers 8 --b2-hard-delete

# 3. 数据库快照上传
rclone copy "$BACKUP_DIR/db-$DATE.dump.gz" "$B2_REMOTE/db/" --transfers 2

# 4. 清本地 + 远端保留策略
find "$BACKUP_DIR" -name 'db-*.dump.gz' -mtime +2 -delete
rclone delete "$B2_REMOTE/db/" --min-age 30d
```

cron：

```cron
30 2 * * * chiyan /usr/local/bin/chiyan-backup.sh >> /var/chiyan/logs/backup.log 2>&1
```

### 6.2 恢复演练（每季度一次，staging）

```bash
# 1. 从 B2 拉最近一份
rclone copy b2:chiyan-backups/db/db-20260528-0230.dump.gz /tmp/

# 2. 灌库
gunzip -c /tmp/db-20260528-0230.dump.gz \
  | sudo -u postgres pg_restore -d chiyan_staging --clean --if-exists

# 3. 媒体回灌
rclone sync b2:chiyan-backups/media/ /var/chiyan/media-staging/

# 4. 跑 smoke test，验证 RPO ≤ 24h / RTO ≤ 2h
```

### 6.3 加密密钥保管

- `/etc/chiyan/secrets.env`（含 AES 主密钥、JWT 私钥、DB / Redis 密码）由主理人 1Password 主条目保管
- 备份脚本**不**把 `secrets.env` 推 B2；VPS 损毁时手动从 1Password 还原
- 密钥轮换流程：写入 `AES_KEY_V2`，应用同时双写（读 v2 写 v2，遇 v1 数据按 v1 解密），三周后下线 v1

## 七、监控

| 维度 | 工具 | 告警阈值 |
| --- | --- | --- |
| 应用错误 | Sentry | error rate > 1% / 5min |
| 接口耗时 | Sentry transactions + Caddy access log | P95 > 500ms / 5min |
| 资源 | node-exporter 风格的简易脚本写 Sentry custom event | CPU > 80% / mem > 1.7G / disk > 30G / 持续 5min |
| 慢查询 | PostgreSQL `log_min_duration_statement=500ms` | journalctl 抓 `duration:` 推 Sentry |
| systemd | `systemctl is-failed` 5 分钟巡检 | 任一 unit failed → 推 Sentry + 邮件 |
| 健康检查 | `curl https://api.chiyan.com/health` 边缘 + 本机 | 连续 3 次失败 → 自动 `systemctl restart chiyan-api` 一次 + 告警 |

不引入 Grafana / Prometheus，2G 机器装不下；Sentry 免费层够用，复杂度低。

## 九、故障预案

| 场景 | 初步动作 | 二级动作 |
| --- | --- | --- |
| `chiyan-api.service` failed | `journalctl -u chiyan-api -n 200`，看错误 → 改 + 重启 | 软链回 `api-prev`，重启 |
| Postgres 内存吃紧 / OOM | `journalctl -u postgresql`、`free -m`；降 `shared_buffers` 至 192M 重启 | 关 Sentry 上传一段时间，腾内存；联系业主停服扩内存 |
| Redis OOM | 检查 maxmemory-policy 是否生效；清非关键 key | 临时下调限流窗口，让请求穿透到 DB |
| 磁盘超 30G | `du -h /var/chiyan/media \| sort -h \| tail`，确认热点；归档 originals 老照片到 B2 | 把 originals 全量挂载移到 B2 + 本地保留 thumbnails |
| Caddy 证书失效（Origin CA 15 年应该不会，但万一） | 临时切到 Let's Encrypt，Cloudflare 改 Full(strict) | 重新签 Origin CA，回切 |
| Cloudflare 边缘故障 | DNS 临时关 proxied，源 IP 暴露 + 仅限白名单 IP 访问度过故障 | 评估迁移到双 CDN |
| 整机宕机 / 主机商挂 | 新开 VPS → 跑 §二 初始化 → 从 B2 恢复 → DNS 切 A 记录 | RPO ≤ 24h, RTO ≤ 4h |
| 被发现 / 域名指向引爆 | 立刻 Cloudflare 改 paused / hold；评估换域名 + 换 IP | 全量重签 URL 前缀；通知业主走 1Password 重发邀请链 |

## 十、不做与边界

- ❌ **不使用 Docker**（业主 2026-05-29 明确决策；apt 原生包 + systemd 已足够，避免容器化的额外内存税）
- ❌ **不引入 k8s / Nomad / Swarm**（单机部署不需要编排）
- ❌ **不接入 Grafana / Prometheus**（2G 机器装不下；Sentry + journalctl 足矣）
- ❌ **不接入 ELK / Loki**（结构化 journal + Caddy access log JSON 直接 `jq` 检索）
- ❌ **不接 Cloudflare Workers / Pages / R2**（v3 推翻原因详见 [API 文档 §10.3](./后台管理系统接口设计方案.md)）
- ❌ **不接内容审核 / 备案 / 短信 / 邮件**（业主决策一致：邀请制 + 海外 + 零 PII）
- ⚠️ 大陆访问 best-effort，不签 SLA；高峰期可能慢、个别 ISP 偶发 RST
